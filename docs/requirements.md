## 1. 背景と目的

- 自社ECサイトの配送に日本郵便のクリックポストを使用している。
- クリックポストの宛名ラベルPDFは **A4の左上固定** で印字されるため、普通紙に印刷して毎回はさみで切り取る運用になっていた。
- ラベルシール用紙 **A-one 77220**(A4・4面・貼るだけ)を導入するにあたり、印字位置をシール用紙の「空いている面」に移動できるツールが必要になった。
- 併せて、今後も社内ツールを同じアプリ内に追加していける「ツール集サイト」の土台を作る。

## 2. 調査結果(前提事実)

### 2.1 クリックポストPDFの仕様

実際に出力されたPDFの解析結果:

- 用紙: A4縦 (210×297mm)、1ページ
- ラベル領域: ページ **左上のちょうど1/4 = 105×148.5mm**(切り取り線・はさみマークがこの境界線上にある)
- ラベル本体(赤枠)は 1/4 領域の内側に収まっており、周囲に若干の余白がある
- ファイル名の形式: `CP<日付8桁>_<宛名>様.pdf`
- **宛先住所・氏名などの個人情報を含む** → リポジトリへのコミット厳禁(§11)
- **PDF は RC4 暗号化されている**(閲覧自由・権限制限のみ)。このため pdf-lib ではページ内容を再利用できず、実装はレンダリング方式を採る(§2.3、implementation-notes.md)

### 2.2 シール用紙 A-one 77220

[製品ページ](https://www.a-one.co.jp/product/search/detail.php?id=77220)

- A4判(210×297mm)、**4面(2列×2段)、1面 105×148.5mm、余白なしの十字分割**
- マット紙・ホワイト、キレイにはがせるタイプ
- インクジェット(染料・顔料)/レーザー/コピー機対応(顔料インクは乾きにくい旨の注意あり)
- クリックポスト対応を謳う製品

### 2.3 結論

クリックポストのラベル面(105×148.5mm)と 77220 の1面は**完全に一致**する。
したがってコア処理は「**元PDFの左上1/4領域を、出力A4の4象限のいずれかへ原寸で配置し直す**」こと。拡大縮小・回転は不要。

実現方式(2026-07-03 実PDF検証により確定): 元PDFが暗号化されているためベクターのままの埋め込みは不可能で、**pdfjs でラベル領域を 600dpi でレンダリングし、画像として pdf-lib で原寸配置する**(バーコード・QR とも印刷品質は十分)。詳細は [implementation-notes.md](implementation-notes.md)。

## 3. 確定要件(ヒアリング結果: 2026-07-03)

| # | 項目 | 決定 |
|---|------|------|
| 1 | 配置UI | **4分割スロット選択式**。プレビュー上の4象限(左上/右上/左下/右下)をクリックして配置先を指定 |
| 2 | 複数配置 | **対応する**。複数のクリックポストPDFを読み込み、1枚のA4に最大4件を面付けして一括出力 |
| 3 | 入力形式 | **1件ずつのPDF(左上1面)のみ**。クリックポストの「まとめ印字」PDF(4面付け・複数ページ)は対象外(§14) |
| 4 | 印字ズレ補正 | **不要**。ラベルは切り取り線内側に余白があるため多少のズレは許容 |

## 4. 技術・構成の決定事項(2026-07-03 ユーザー承認済み)

| # | 項目 | 決定 | 備考 |
|---|------|--------|------|
| 1 | UIフレームワーク | **React** | React Router でホーム+ツールのルーティング |

## 5. 機能要件

### 5.1 ツール①: クリックポスト面付けツール(slug: `clickpost-label`)

※ツールの正式名称は未定(§13)。

#### 入力

- PDFファイルを最大4件読み込む(**ドラッグ&ドロップ + ファイル選択ダイアログ**の両対応)
- 読み込み時の検証:
  - PDF形式であること
  - ページサイズがA4(±数mmの誤差許容)であること → 違う場合は警告
  - 複数ページのPDFは先頭ページのみ使用し、その旨を表示
- 5件以上読み込もうとした場合: 4件で打ち切り、警告表示(複数ページ出力はスコープ外 §14)
- ファイル名(`CP<日付8桁>_<宛名>様.pdf` 形式)を一覧に表示し、どの宛名かわかるようにする

#### 配置

- 画面にA4のプレビュー(4スロットのグリッド)を表示
- 読み込んだPDFは **読み込み順に 左上→右上→左下→右下 へ自動配置**し、その後クリック操作で自由に移動・入れ替え・解除できる
  - 例: 1件だけ読み込んで「右下」へ移動(用紙の左上が使用済みのケース)
- プレビューには **pdfjs-dist でラベルの実画像を描画**する(ファイル名だけでなく実物を見せて誤配置を防ぐ)

#### 出力

- pdf-lib で新規A4ページを生成し、各入力PDFの左上1/4領域(105×148.5mm)をクロップ付き `embedPage` で各スロット位置へ**原寸配置**(§10 座標仕様)
- **「PDFをダウンロード」**: ファイル名は `clickpost_<日時>.pdf` 形式
- **「印刷」**: 生成PDFをブラウザの印刷ダイアログで直接印刷(hidden iframe 方式)
- 印刷時の注意書きを画面に常設表示: 「**実際のサイズ(倍率100%)で印刷してください**」(拡大縮小されるとシール枠とずれるため)

### 5.2 ホーム画面

- ヘッダー(サイト名)+ ツールカードの一覧(ツール名・説明・アイコン)
- カードをクリックすると各ツール画面へ遷移
- **ツールレジストリ方式**: `src/tools/registry.ts` にメタ情報(名前・説明・パス・コンポーネント)を1件追加するだけで、ホーム画面とルーティングに自動反映される構成にする(将来のツール追加を容易に)

## 6. 非機能要件

| 項目 | 内容 |
|---|---|
| 処理方式 | **完全クライアントサイド処理**。PDFを外部サーバーへ一切送信しない(GitHub Pages は静的配信のみ)。宛名という個人情報を扱うため必須要件とする |
| 対応ブラウザ | 最新の Chrome / Edge / Safari / Firefox。**デスクトップ優先**(印刷作業はPCから行う想定) |
| 言語 | 日本語のみ |
| オフライン動作 | 必須ではない(将来のPWA化は拡張候補) |

## 7. 技術スタック

| 領域 | 採用 | 理由 |
|---|---|---|
| ビルド | Vite | ユーザー指定 |
| 言語 | TypeScript | ユーザー指定 |
| CSS | Tailwind CSS | ユーザー指定 |
| UI | React(仮決定 §4) | エコシステム最大、将来のツール追加に有利 |
| ルーティング | React Router(**HashRouter**) | GitHub Pages でのリロード404問題を確実に回避。URLは `/#/tools/clickpost-label` 形式。<br>※`BrowserRouter` + 404.html リダイレクトハックという代替もあるが、社内ツール用途ではハッシュ方式が単純確実 |
| PDF読み込み・描画 | **pdfjs-dist** | 暗号化PDFの復号と canvas レンダリング(ラベルの画像化+プレビュー)。cmaps 等の同梱アセットを postinstall で public/pdfjs へコピーして配信 |
| PDF生成 | **pdf-lib** | 新規A4に `embedPng` + `drawImage` でラベル画像を原寸配置 |
| デプロイ | GitHub Actions → GitHub Pages(§4) | push だけで公開 |

## 10. 座標仕様(実装リファレンス)

- PDF座標系は**左下原点**、単位 pt(1mm = 72/25.4 ≈ 2.83465pt)
- A4 = 595.28 × 841.89 pt / 1面 = 297.64 × 420.945 pt

| スロット | 原点(左下basis) x, y (pt) |
|---|---|
| 左上 | (0, 420.945) |
| 右上 | (297.64, 420.945) |
| 左下 | (0, 0) |
| 右下 | (297.64, 0) |

- 元PDFからのクロップ領域(embedPage の boundingBox): `x: 0〜297.64, y: 420.945〜841.89`(= 左上1/4)
- 配置は原寸(scale = 1.0)。拡大縮小・回転なし

## 11. セキュリティ / プライバシー

- 実際のクリックポストPDF(宛名入り)は**リポジトリにコミットしない**。`.gitignore` に `*.pdf` を追加する
- 動作確認用にはダミーデータのPDFを別途生成して使う
- アプリはPDFをブラウザ内でのみ処理し、外部送信しない(§6)
- GitHub Pages の無料利用は Public リポジトリが前提(Free プランの場合)。公開されるのはコードのみで問題ない構成とする

## 15. v2 要件(2026-07-03 承認)

### 15.1 プロジェクトの再定義

| 項目 | 決定 |
|---|---|
| 名称 | **clickpost-tool**(クリックポスト面付けツール専用アプリ) |
| リポジトリ | `r77tchan/clickpost-tool`(Public)を新規作成。**旧 skym-tools リポジトリと旧 Pages はそのまま残す**(無効化しない) |
| トップページ | **廃止**。開いたら即ツール画面(ルーティング・ツールレジストリは削除) |
| ローカルフォルダ名 | 変更しない(リモートがあればよく、ローカルは後日削除予定) |

### 15.2 切り取り線

| 項目 | 決定 |
|---|---|
| 元PDFの切り取り線(右端・下端の破線+ハサミ) | **常に除去**する。実装はレンダリング後に該当帯(右端・下端 各4mm)を白塗り。**ラベルの位置・サイズは一切変えない**(点線部分以外の表示は現状維持) |
| 自前の切り取り線 | A4を必ず4分割する**十字の破線**(薄グレー、端から端まで)を出力PDFに描画 |
| ON/OFF | チェックボックスで切替、**初期値OFF** |

### 15.3 配布形態(2系統ビルド)

| 形態 | コマンド | 内容 |
|---|---|---|
| GitHub Pages | `npm run build` | `https://r77tchan.github.io/clickpost-tool/`。main へ push で自動デプロイ |
| シングルHTML | `npm run build:single` | `dist-single/index.html` **1ファイル**(約2.4MB)。全アセット埋め込みで、**PCの Chrome/Edge でダブルクリック(file://)するだけで動作**。ネットワーク接続不要(外部リクエストゼロを実測確認済み) |
